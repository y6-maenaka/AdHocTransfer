[ 概要 ]
トランスポート層プロトコルにUDPを使ったファイル転送ソフト


[ 条件 ]
1, 相手のIP(グローバル)を指定してファイルを転送する
2, 指定のディレクトリに転送するファイル(FILE)を配置する
3, FILEは指定byteに細切れ(BLOCK)にする
4, 各BLOCKには自身のBLOCKのハッシュ値とシーケンス番号、送信元情報、全体情報を保持したヘッダをつけて送信する 
5, ヘッダをつけ終わったBLOCKは簡易暗号化する
6, 受信側はBLOCKのボディのハッシュ値を求めてヘッダのハッシュ値と比較する　Falseだった場合はデータを破棄する(パリティチェック)
7, ヘッダを外し受信したファイルを結合する　この際に欠損しているデータのシーケンス番号を送信元に送信しデータを要求する
8, 破損データを要求されたホストは破損データを送信する
9, TCP（受信元から全てのデータが元ファイルを復元できたら完了信号を送信する）
10, 完了信号を受信するまではBLOCKとしてデータを保存する
11, 受信側は破損ファイルを検出した時点で送信元に欠損ファイル情報を送信し、欠損ファイル要求信号を受け取った送信元はすぐに送り返す

[ 条件2 ]
1, 中止信号をシグナルとして送受信できる
2, 受信側は破損ファイルを検出した時点で送信元に欠損ファイル情報を送信し、欠損ファイル要求信号を受け取った送信側はすぐに送り返す
3, ファイル送受信、ファイルチェック、ファイル結合シーケンスに主な処理は分かれているものとする




ファイル匿名公開アプリ
・他者のファイルを閲覧しするためにはこのソフトを起動する必要があり、このソフトが起動しているときは他のノードと通信できるようにする。
・ファイルは参加者内で分散して保持され、元が特定できないようにする
・ファイルは暗号化され、ブロック単位で保持される

[命名規則]
１、ファイル名はPascalCase
２、関数名はcamelCase
３、変数名はcamelCase
